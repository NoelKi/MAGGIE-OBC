📊 MODULE ÜBERSICHT

I2C Bus (hal_i2c)
  → Geräte: IMU (0x68/0x69), Barometer (0x76/0x77)
  → Funktionen: Init, Write, Read, WriteRead, Scan
  → Frequenz: 400 kHz

SPI Bus (hal_spi)
  → Geräte: Drucksensor (MS5607), SD-Card
  → Funktionen: Init, Transfer, Write, Read, RegRead, RegWrite
  → Frequenz: 10 MHz

CAN Bus (hal_can) [erweitert]
  → Geräte: STM32 Roboterarm
  → Funktionen: Init, Send, Receive
  → Baudrate: 500 kbps

ADC (hal_adc)
  → Kanäle: A0, A1 (Gewichtssensoren)
  → Auflösung: 8, 10, 12, 14 Bit
  → Funktionen: Read, Average, ToVoltage, Calibrate

PWM Motor (hal_pwm)
  → Motor: HDRM mit H-Bridge
  → Funktionen: SetSpeed, SetDirection, Enable, EmergencyStop
  → Richtungen: Forward, Backward, Stop, Brake

UART Serial (hal_uart)
  → Verwendung: Telemetry, Debug
  → Baudrate: 115200 bps
  → Funktionen: Write, Read, WriteString, WriteData

🚀 SCHNELLSTART

In main.cpp:

  #include "hal/hal.h"

  void setup() {
      // Initialisiere alle HAL-Module auf einmal
      if (!HAL::initializeAll()) {
          while(1) delay(1000);  // Error
      }
      
      Serial.println("All systems ready!");
  }

  void loop() {
      // Nutze HAL-Funktionen
      uint16_t weight = HAL::adcRead(HAL_ADC_SCALE1);
      HAL::pwmSetMotor(50, HAL::MOTOR_FORWARD);
  }

✨ HARDWARE-WECHSEL

Mit HAL ist Hardware-Migration ein Eintageswerk!

Beispiel: Teensy 4.1 → STM32H7

  Ohne HAL: ~500 Zeilen Code durchsuchen & anpassen (fehler-anfällig)
  Mit HAL:  Nur hal/*.cpp Implementierungen umschreiben (sauber)

Alle Services/Drivers bleiben unverändert!

📋 VERWENDUNG IN SERVICES

Beispiel: IMUDriver nutzt HAL

  #include "hal/hal.h"

  bool IMUDriver::init() {
      if (!HAL::i2cInit()) return false;
      HAL::i2cScan();
      
      uint8_t whoAmI;
      return HAL::i2cWriteRead(0x68, 0x75, &whoAmI, 1);
  }

  void IMUDriver::readData() {
      uint8_t buffer[6];
      HAL::i2cRead(0x68, buffer, 6);
  }

✅ VORHER vs. NACHHER

VORHER (ohne HAL):
  imu_driver.cpp:
    digitalWrite(13, HIGH);
    Wire.beginTransmission(0x68);
    Wire.write(0x3B);
    Wire.endTransmission();
    Wire.requestFrom(0x68, 6);

NACHHER (mit HAL):
  imu_driver.cpp:
    HAL::setLed(true);
    HAL::i2cWriteRead(0x68, 0x3B, buffer, 6);

  Vorteile:
    ✅ Lesbar
    ✅ Wartbar
    ✅ Hardware-unabhängig
    ✅ Zentrale Konfiguration
